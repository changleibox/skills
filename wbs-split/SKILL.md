# 智能网页需求 WBS 拆分工具

## 概述

通过 **Qoder AI 辅助**从任意网页提取表格数据并进行WBS拆分的智能工具。

**核心特点**：
- 🤖 **AI 协同工作** - 无需安装浏览器，由 Qoder AI 负责读取网页内容
- 🔥 **蓝湖深度集成** - 自动调用 lanhu MCP 工具精准提取蓝湖需求数据
- 🌐 **通用兼容性** - 支持 Jira、Confluence、Excel Online 等任意表格页面
- 📦 **零额外依赖** - 仅需 openpyxl，无需 Playwright 或其他浏览器工具
- 🎯 **强制平台过滤** - 默认只生成 APP 端需求，避免混入无关内容
- ⏱️ **任务粒度控制** - 强制验证任务工时不超过 3 小时，确保任务可执行

## ⚠️ 重要约束（严格执行）

### 1. 平台过滤约束
- **默认行为**：`--target-platform=app`，只生成 APP 端需求的 WBS
- **强制要求**：除非明确指定 `--target-platform=pc` 或 `all`，否则必须过滤掉 PC 端内容
- **过滤规则**：排除包含"PC"、"后台管理"、"管理后台"等 PC 端关键词的任务

### 2. 任务粒度约束
- **默认行为**：`--max-task-hours=3`，单个任务不超过 3 小时
- **强制要求**：所有任务的预估工时必须 ≤ max_task_hours 设定值
- **验证机制**：工具自动检测超限任务并输出警告，要求拆分
- **粒度建议**：
  - 1-2 小时：简单 UI 组件、样式调整、单一功能点
  - 2-3 小时：带状态管理的页面、复杂表单、列表交互
  - 超过 3 小时：必须拆分为多个子任务

支持：蓝湖、Jira、Confluence、Excel Online等任意包含表格的网页。

## 核心功能

1. **AI 协同工作** - Qoder AI 负责读取网页内容，工具负责数据处理
2. **蓝湖深度集成** - 自动识别蓝湖页面，调用 lanhu MCP 工具获取数据
3. **通用网页支持** - AI 使用 fetch_content 读取其他网页的表格内容
4. **智能筛选** - 根据模块名称过滤和保留指定内容
5. **需求映射** - 自动填充需求名称列
6. **Excel导出** - 生成标准WBS格式文件

## 使用方法

### 两步工作流

#### 步骤 1：运行工具获取指引

```bash
# 运行工具，会显示操作指引
/run wbs-split --url "网页链接"
```

工具会根据URL类型（蓝湖/其他）显示不同的指引。

#### 步骤 2：让 AI 获取数据

**对于蓝湖页面：**
```
请使用 lanhu_get_ai_analyze_page_result 工具读取这个蓝湖页面的表格数据
```

**对于其他网页：**
```
请使用 fetch_content 工具读取这个网页的表格数据，并提取为JSON格式
```

#### 步骤 3：使用数据文件重新运行

```bash
# AI 获取数据后，使用 --data-file 参数运行
/run wbs-split \
  --url "网页链接" \
  --data-file table_data.json \
  --output "需求_WBS.xlsx" \
  --keep-modules "模块1,模块2" \
  --demand-map "模块1:需求1,模块2:需求2"
```

### 完整示例：蓝湖审核中心

```bash
# 步骤 1：运行工具获取指引
/run wbs-split \
  --url "https://lanhuapp.com/web/#/item/project/product?tid=...&docId=..."

# 步骤 2：按照提示让 AI 获取数据
# AI 会调用 lanhu_get_ai_analyze_page_result 工具

# 步骤 3：使用数据文件完成拆分（默认只生成APP端，任务粒度≤3h）
/run wbs-split \
  --url "https://lanhuapp.com/web/#/item/project/product?tid=..." \
  --data-file lanhu_table_data.json \
  --output "审核中心_WBS.xlsx"

# 可选：生成PC端需求
/run wbs-split \
  --url "https://lanhuapp.com/web/#/item/project/product?tid=..." \
  --data-file lanhu_table_data.json \
  --target-platform pc \
  --output "审核中心_PC_WBS.xlsx"

# 可选：调整任务粒度为2小时
/run wbs-split \
  --url "https://lanhuapp.com/web/#/item/project/product?tid=..." \
  --data-file lanhu_table_data.json \
  --max-task-hours 2 \
  --output "审核中心_细粒度_WBS.xlsx"
```

## 参数说明

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| --url | 是 | - | 需求页面URL（用于识别页面类型） |
| --data-file | 否 | - | 包含表格数据的JSON文件路径 |
| --output | 否 | ./需求_WBS.xlsx | 输出Excel文件路径 |
| **--target-platform** | **否** | **app** | **目标平台：app（仅APP端）、pc（仅PC端）、all（全部平台）** |
| **--max-task-hours** | **否** | **3** | **单个任务的最大工时（小时），用于粒度控制** |
| --keep-modules | 否 | 全部 | 保留的模块列表（逗号分隔） |
| --exclude-modules | 否 | 默认排除列表 | 排除的模块列表（逗号分隔） |
| --demand-map | 否 | 无 | 模块到需求名称的映射 |

**⚠️ 重点参数**：
- `--target-platform`：**默认为 app**，确保只生成 APP 端需求
- `--max-task-hours`：**默认为 3 小时**，强制任务粒度控制

## 输出格式

Excel 文件包含以下列：

- **需求名称**：通过 `--demand-map` 参数映射填充（如不指定则为空）
- **模块（按需求模块归类）**：保持原始文档的模块名称，**不添加编号前缀**
- **任务**：保持原始文档的任务名称，**不添加编号前缀**
- **时间**：纯数字（小时），如 `2`、`3`、`1.5`（不带单位）
- 拆解者
- 进度
- **剩余时间**：纯数字（小时），如 `2`、`3`、`1.5`（不带单位）
- 是否Flutter开发
- 是否H5开发
- 是否可用AI完成

**⚠️ 重要约定**：
1. **需求名称**：必须使用原始需求文档中的名称（如"审核中心"、"批量审核"），禁止添加 R001 等编号
2. **模块名称**：保持 AI 从文档中提取的原始名称（如"审核中心主页"），禁止添加 M1、M2 等前缀
3. **任务名称**：保持 AI 从文档中提取的原始名称（如"TabBar框架搭建"），禁止添加 M1-1、M2-3 等编号
4. **时间字段**：只填写纯数字（单位为小时），如 `2`、`3`、`1.5`，禁止添加 "h"、"小时" 等单位

## 常见问题

### Q: 为什么需要两步操作？

**原因**：工具本身不能直接访问网页，需要 Qoder AI 的帮助

**优势**：
- ✅ 无需安装浏览器和 Playwright
- ✅ AI 可以智能处理登录、验证码等问题
- ✅ 支持蓝湖 MCP 工具的深度集成
- ✅ 更灵活的数据提取方式

### Q: JSON 数据文件格式？

**格式要求**：
```json
{
  "headers": ["需求名称", "模块", "任务", "时间", ...],
  "data": [
    {
      "需求名称": "",
      "模块": "审核中心主页",
      "任务": "TabBar框架搭建",
      "时间": "2",
      "剩余时间": "2",
      ...
    },
    ...
  ]
}
```

**⚠️ 重要提醒**：
- **模块**和**任务**字段必须使用原始文档中的名称
- **禁止添加** "M1"、"M2-1" 等编号前缀
- **时间**和**剩余时间**字段只填写纯数字（单位：小时），如 `"2"`、`"3"`、`"1.5"`
- **需求名称**字段可以为空，后续通过 `--demand-map` 参数填充

**AI 会自动生成符合要求的 JSON 文件**

### Q: AI 无法访问蓝湖页面？

**解决方案**：
- 确保蓝湖URL完整且可访问
- 检查是否需要登录（AI 可能需要你手动登录后分享Cookie）
- 尝试先手动导出蓝湖表格为 Excel，再让 AI 读取

### Q: 其他网页数据不完整？

**原因**：某些网页使用动态加载，fetch_content 可能只能获取初始内容

**解决方案**：
- 让 AI 多次尝试，或者使用不同的查询参数
- 手动复制网页表格内容，让 AI 转换为 JSON
- 导出为 Excel 后让 AI 读取

## 技术细节

- **协同模式**：Qoder AI + Python 工具
- **AI 能力**：
  - 蓝湖：lanhu MCP 工具（lanhu_get_ai_analyze_page_result）
  - 其他：fetch_content 工具
- **数据格式**：JSON 中间格式
- **数据处理**：实时过滤 + 映射 + 平台过滤 + 粒度验证
- **导出格式**：标准 Excel (xlsx)
- **依赖项**：仅 openpyxl

### 强制约束机制

#### 1. 平台过滤
- **默认值**：`target_platform = "app"`
- **过滤逻辑**：
  - APP 端：排除包含 "PC"、"后台管理"、"管理后台" 等关键词的任务
  - PC 端：只保留包含上述关键词的任务
  - 全部：不进行过滤
- **实现位置**：`filter_by_platform()` 函数

#### 2. 任务粒度验证
- **默认值**：`max_task_hours = 3.0`
- **验证逻辑**：
  - 解析时间字符串（支持 "2h"、"3小时"、"1天" 等格式）
  - 转换为小时数（1天 = 8小时）
  - 检测超过限制的任务
  - 输出警告信息和违规任务列表
- **实现位置**：`validate_task_granularity()` 函数
- **处理方式**：
  - 超限任务仍会输出到 Excel
  - 但会在控制台显示警告
  - 提示用户手动拆分超限任务

### 支持的数据来源

**蓝湖页面**：
- 通过 lanhu MCP 工具直接读取
- 支持所有蓝湖原型图表格

**其他网页**：
- 通过 fetch_content 工具读取
