# 多Bug处理模式详细流程

> 本文件是 ishop-yunxiao-bug-fix 技能的子文件（v2.3.0），包含步骤三B~三D的详细操作流程。
> 由主文件 SKILL.md 在多Bug处理场景中通过 read_file 引用。

### 步骤三B：批量操作模式

**1. 确认选中列表**

展示所有选中的Bug：
```
已选中 N 个Bug进行批量处理：
1. [BUG-XXX] 标题... | 优先级：高
   🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/{identifier}
2. [BUG-XXX] 标题... | 优先级：中
   🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/{identifier}
...
```

**2. 选择统一操作**

使用 `AskUserQuestion` 选择操作方式：
- [批量修复] - 统一提交修复说明并更新状态为已修复
- [批量转交] - 统一转交给指定负责人
- [批量标记无法复现]
- [批量标记不是问题]
- [批量标记需要更多信息]
- [批量标记重复Bug]
- [批量延期处理]
- [批量更新状态] - 仅更新状态，不提交评论

**3. 填写统一模板**

根据选择的操作方式，展示对应的评论模板（与单Bug处理使用相同模板）。
用户填写一次，应用到所有选中的Bug。

特殊处理：
- 批量修复：修复说明中的"问题原因"和"修改描述"应用到所有Bug
- 批量转交：询问统一的转交目标人（使用智能推荐或手动输入）
- 批量更新状态：查询 `mcp_RAM_get_work_item_workflow` 获取可用状态供选择

**4. 批量执行**

逐个调用API处理每个Bug，展示实时进度：
```
批量处理进度：
✅ [1/N] BUG-001 - 已完成
✅ [2/N] BUG-002 - 已完成
⏳ [3/N] BUG-003 - 处理中...
```

每个Bug的处理步骤：
1. 调用 `mcp_RAM_create_work_item_comment` 提交评论（如需要）
2. 调用 `mcp_RAM_update_work_item` 更新状态/负责人

**5. 批量结果摘要**

```
===== 批量处理结果 =====
操作类型：批量修复
处理总数：N 个
成功：X 个
失败：Y 个

失败明细（如有）：
- [BUG-XXX] 失败原因：...
========================
```

**6. 错误处理**

- 单个Bug处理失败时：记录错误，继续处理下一个（不中断批量流程）
- 所有Bug处理完成后：汇总展示失败项，询问是否重试失败项

处理完毕后，将所有处理过的Bug记录到"处理记录表"，回到步骤三继续处理剩余Bug。

### 步骤三C：流水线模式（逐个精简处理）

按顺序处理每个选中的Bug，采用精简流程加速处理。

**流水线循环**：

对于每个选中的Bug（第 X/N 个）：

1. **精简详情展示**

   调用 `mcp_RAM_get_work_item` 获取详情，展示摘要：
   ```
   ━━━ 第 X/N 个 ━━━
   [BUG-XXX] 标题
   优先级：高 | 状态：待处理 | 创建人：张三
   🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/{identifier}
   描述：（前200字）...
   ━━━━━━━━━━━━━━━
   ```

2. **快速选择处理方式**

   使用 `AskUserQuestion`：
   - [修复此Bug]
   - [转交给其他人]
   - [无法复现]
   - [不是问题/设计如此]
   - [需要更多信息]
   - [重复Bug]
   - [延期处理]
   - [跳过此Bug]
   - [结束流水线，跳过剩余]

3. **快速填写模板**

   根据选择展示对应模板。
   提供快捷选项：[复用上一个Bug的模板内容]（如果不是第一个Bug）

4. **提交并继续**

   提交评论 + 更新状态后，自动进入下一个Bug（无需回到列表选择）。

**流水线结束**：
- 所有Bug处理完毕，或用户选择"结束流水线"
- 展示本轮流水线处理摘要
- 将处理记录写入"处理记录表"，回到步骤三继续处理剩余Bug

### 步骤三D：智能分组处理

**1. AI 分析与分组**

对所有选中Bug调用 `mcp_RAM_get_work_item` 获取标题和描述，然后：
- 提取每个Bug标题/描述中的模块关键词（如"开单"、"库存"、"客户"、"报表"等）
- 按关键词相似度聚类分组
- 无法归类的Bug归入"未分组"

**2. 展示分组结果**

```
智能分组结果（共 N 个Bug，分为 M 组）：

第1组 - 开单相关（3个）：
  • [BUG-001] 销售开单页面选择客户时闪退
    🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/BUG-001
  • [BUG-003] 开单时商品数量无法输入小数
    🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/BUG-003
  • [BUG-007] 开单页面价格计算错误
    🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/BUG-007

第2组 - 库存相关（2个）：
  • [BUG-002] 库存盘点数量显示异常
    🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/BUG-002
  • [BUG-005] 入库单审核后库存未更新
    🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/BUG-005

未分组（1个）：
  • [BUG-004] 系统设置页面打开缓慢
    🔗 https://devops.aliyun.com/projex/project/{spaceId}/bug/BUG-004
```

使用 `AskUserQuestion` 确认：
- [确认分组，开始处理]
- [调整分组]（用户通过 Other 输入调整指令）

**3. 逐组处理**

对每一组：
1. 展示组内Bug列表
2. 使用 `AskUserQuestion` 选择该组的统一操作方式（同步骤三B的操作选项）
3. 填写对应评论模板（一次填写，应用到组内所有Bug）
4. 批量执行（同步骤三B的批量执行逻辑）
5. 展示该组处理结果

**4. 未分组Bug处理**

未分组的Bug按**流水线模式**（步骤三C）逐个处理。

**5. 分组处理完成**

展示整体分组处理汇总：
```
===== 分组处理汇总 =====
总计：N 个Bug，M 个分组

第1组（开单相关）：3个 → 批量修复 ✅
第2组（库存相关）：2个 → 批量转交给李四 ✅
未分组：1个 → 延期处理 ✅
=========================
```

将所有处理记录写入"处理记录表"，回到步骤三继续处理剩余Bug。
